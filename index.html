Setup, Deployment & Monitoring Guide. v1.1.0

This repository provisions, configures, and deploys a small web + monitoring stack on AWS using Terraform and Ansible. It serves as a proof-of-concept for secure, private networking with public exposure via load balancers and tunnels.

1. Architecture Overview

All infrastructure is provisioned in the ap-southeast-1 (Singapore) region.

  • Infrastructure: Terraform (VPC, Subnets, EC2, IAM, ECR).
  • Configuration: Ansible (Docker installation, App deployment, Monitoring setup).
  • Application: Dockerized web app with custom environment variables.
  • Monitoring: Prometheus & Grafana (node_exporter).

2. Quick Start

Ensure you have terraform, ansible, and aws-cli installed.

Environment Setup

aws configure --profile default
export AWS_REGION=ap-southeast-1

3. Dev Diary: Challenges & Solutions

Real-world battles fought during this deployment.

Challenge 1: The "AWS Not Found" Error

Problem: Ansible failed at ECR Login because the vanilla Ubuntu AMI lacks the AWS CLI.
Fix: Updated playbook to install dependencies first.

- name: Download AWS CLI
  get_url:
    url: "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" ...

- name: Install AWS CLI
  command: /tmp/aws/install --update

Challenge 2: The "Stale Image" Persistency

Problem: The Web Server refused to show the new version of the app (e.g., name changes) and stubbornly stuck to the old Docker image, even after a successful pipeline run.
Fix: The "Fresh Start" Protocol. The caching on both local and remote ends was too aggressive. I had to explicitly delete the local image to force a new build, and remove the container on the instance.

# 1. Local: Delete old image & rebuild
docker rmi 
docker build -t my-app .
docker push ...

# 2. Remote: Stop & Remove old container
docker stop my-app
docker rm my-app
# (Then run Ansible to pull the fresh image)

Challenge 3: The "Localhost" Monitoring Trap

Problem: Grafana running in Docker couldn't reach Prometheus via localhost:9090.
Fix: Configured Grafana to use the Private IP of the host server.

http://10.0.0.136:9090

4. How to Run

Step 1: Provision Infrastructure

cd terraform
terraform init
terraform plan -out=plan.tfplan
terraform apply "plan.tfplan"

Step 2: Configure Servers (Ansible)

Run these from the Ansible Controller node.

# 1. Install Docker
ansible-playbook -i inventory.ini install_docker.yml

# 2. Deploy Web App
ansible-playbook -i inventory.ini deploy_web.yml

# 3. Setup Monitoring
ansible-playbook -i inventory.ini deploy_monitor.yml
